# Minimal Programmer Blog for GitHub Pages

This repository is a plain static site designed for GitHub Pages.

## Structure

- `index.html` — home page with intro and latest posts
- `posts.html` — all writing in reverse chronological order
- `tracker.html` — current learning and work
- `projects.html` — plain project list
- `about.html` — longer personal blurb
- `posts/` — individual post pages
- `styles.css` — shared styling

## Publish on GitHub Pages

1. Create a GitHub repository.
2. Push these files to the repository root.
3. In GitHub, open `Settings` -> `Pages`.
4. Under `Build and deployment`, choose `Deploy from a branch`.
5. Select your default branch and `/ (root)`.
6. Save and wait for Pages to publish.

If you use a repository site instead of a user site, these relative links will still work as-is.

## Personalization

- Update any remaining copy if you want a different public bio.
- Add more project entries to `projects.html` as they are documented.
- Expand the posts in `posts/` from these imported summaries if you want the full original articles here.
